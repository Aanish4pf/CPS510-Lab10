Tailwind CSS Framework Assignment

Installation Process

Installing Tailwind CSS was super simple since I used the CDN version.

  1. Created a new HTML file called index.html
  2. Added the Tailwind CSS CDN link in the head section: <script src="https://cdn.tailwindcss.com"></script>
  3. Started writing HTML with Tailwind utility classes
  4. Uploaded the file to GitHub repository
  5. Enabled GitHub Pages in repository settings

Building the Page

Building with Tailwind was really straightforward:

  • Used utility classes directly in HTML (like bg-purple-600, text-white, rounded-xl)
  • No need to write custom CSS - everything is done with classes
  • Applied responsive design with prefixes like md: and lg:
  • Used flexbox and grid utilities for layout
  • Added gradients, shadows, and rounded corners easily

Tailwind is a utility-first CSS framework, meaning you style elements by applying pre-made classes instead of writing custom CSS. It makes styling super fast once you learn the class names.

Difficulties and Solutions

Learning Class Names

Problem: Had to keep looking up what class names to use for different styles.

Solution: Kept the Tailwind docs open in another tab. After a while, the pattern made sense (like text-xl for font size, p-4 for padding).

Spacing Issues

Problem: Elements were too close together and looked cramped.

Solution: Used margin classes (mb-4, mt-8) and padding classes (p-6, px-4) to add proper spacing. The number scale (1-12) was easy to understand.

GitHub Pages Not Updating

Problem: Made changes but the live site wasn't updating.

Solution: Had to wait a few minutes for GitHub Pages to rebuild. Also made sure I was pushing to the correct branch (usually main or gh-pages).
